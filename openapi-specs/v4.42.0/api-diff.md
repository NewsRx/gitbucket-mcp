## 3. Schema Changes

### 3.1 Branch Name Length Increase

**Schema:** `Branch.name`  
**Change:** Maximum length increased from 100 to 255 characters  
**Impact:** API consumers can now create branches with longer names (up to 255 characters)

**OpenAPI Specification Update:**
```json
"Branch": {
  "type": "object",
  "properties": {
    "name": {
      "type": "string",
      "maxLength": 255,
      "description": "Branch name (max 255 characters)"
    },
    ...
  }
}
```

**Migration Notes:**
- Clients validating branch names should update validation to accept up to 255 characters
- No changes required for clients that do not validate branch name length
- Backward compatible - all branch names valid in 4.41.0 remain valid in 4.42.0

## 5. Breaking Changes

### 5.1 Java 17 Required

**Breaking Change:** GitBucket 4.42.0 drops Java 11 support. Java 17 is now required.

**Scope:** Server-side deployment only. **No API client impact.**

API clients are not affected by this server-side requirement change.

## 13. Migration Guide

**Minimal migration required:**

For clients that validate branch names:

**Before (4.41.0):**
```python
# Assuming 100 character limit
if len(branch_name) > 100:
    raise ValueError("Branch name too long")
```

**After (4.42.0):**
```python
# Update to 255 character limit
if len(branch_name) > 255:
    raise ValueError("Branch name too long")
```

For clients without branch name length validation: **No changes required.**
